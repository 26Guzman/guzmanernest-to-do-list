Dashboard - Setup Guide

Step 1: Import Database (5 minutes)
Option A: Using phpMyAdmin (Easiest)
  1. Open http://localhost/phpmyadmin/
  2. Click the Import tab
  3. Upload database-structure.sql
  4. Click Go
Option B: Using Command Line
mysql -u root -p < database-structure.sql
Step 2: Configure Database

Edit db-connection.php and update:

define('DB_HOST', 'localhost'); // Your host define('DB_USER', 'root'); // Your username define('DB_PASS', ''); // Your password define('DB_NAME', 'taskflow_db'); // Database name
Step 3: Upload Files

Copy these files to your web folder:

✓ index.php ✓ dashboard.php ✓ create.php ✓ edit.php ✓ db-connection.php

Upload to:

  • XAMPP: C:\xampp\htdocs\your-folder\
  • WAMP: C:\wamp\www\your-folder\
  • Online: /public_html/your-folder/
Step 4: Test the Application

Open in your browser:

http://localhost/your-folder/

You should see:

  • ✅ Statistics cards with task counts
  • ✅ 5 sample tasks in the dashboard
  • ✅ Navigation buttons to Create, View, Edit tasks
File Structure & Purpose
File Purpose
index.php Landing page with statistics & navigation
dashboard.php Main dashboard with task list & delete functionality
create.php Form to add new tasks
edit.php Form to modify existing tasks
db-connection.php Database configuration & connection
database-structure.sql SQL file with table schema & sample data

Troubleshooting
❌ "Connection failed" Error
  • Check MySQL is running
  • Verify database credentials in db-connection.php
  • Check database name is correct
❌ No tasks appearing
  • Verify database-structure.sql was imported
  • Check tasks table exists
  • Verify sample data was inserted
❌ Styling looks broken
  • Check internet connection (Bootstrap is from CDN)
  • Check browser console for CSS errors
  • Try clearing browser cache
❌ Blank white page
  • Enable PHP error reporting
  • Check server error logs
  • Verify PHP version (need 7.4+)
🔒 Security Recommendations
  • Use environment variables for database credentials
  • Enable HTTPS in production
  • Use prepared statements for queries
  • Add user authentication
  • Implement CSRF token protection
  • Keep PHP and dependencies updated
