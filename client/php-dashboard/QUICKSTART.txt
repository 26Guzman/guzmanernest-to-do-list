TASKFLOW DASHBOARD - PHP VERSION
=================================

QUICK START GUIDE (5 Minutes)

STEP 1: Import Database
-----------------------
1. Open phpMyAdmin (http://localhost/phpmyadmin/)
2. Click "Import" tab
3. Choose "database-structure.sql" file from php-dashboard folder
4. Click "Go" to import

OR use command line:
$ mysql -u root -p < php-dashboard/database-structure.sql


STEP 2: Update Database Credentials
------------------------------------
Edit: php-dashboard/db-connection.php

Change these lines to match your MySQL setup:
  define('DB_HOST', 'localhost');      // Your host
  define('DB_USER', 'root');           // Your username
  define('DB_PASS', '');               // Your password
  define('DB_NAME', 'taskflow_db');    // Database name


STEP 3: Upload Files to Web Server
-----------------------------------
Copy these files to your web folder:
  ✅ dashboard.php
  ✅ edit.php
  ✅ create.php
  ✅ db-connection.php

Examples:
  - XAMPP: C:\xampp\htdocs\your-folder\
  - WAMP: C:\wamp\www\your-folder\
  - Online: /public_html/your-folder/


STEP 4: Open in Browser
-----------------------
Navigate to:
  http://localhost/your-folder/dashboard.php

You should see:
  ✅ 3 stat cards (Pending, In Progress, Completed)
  ✅ Table with 5 sample tasks
  ✅ Edit and Delete buttons for each task


FILE STRUCTURE
==============

dashboard.php
  ↳ Main dashboard page
  ↳ Shows task overview and list
  ↳ Handles task deletion
  ↳ Displays success/error messages

edit.php
  ↳ Edit task page
  ↳ Update task title, category, status, etc
  ↳ Requires "?id=1" in URL

create.php
  ↳ Create new task page
  ↳ Add new tasks to database
  ↳ Form validation included

db-connection.php
  ↳ Database configuration file
  ↳ Connection settings
  ↳ DO NOT expose this file publicly in production!

TROUBLESHOOTING
===============

Problem: "Connection failed" error
Solution: Check MySQL is running and credentials are correct

Problem: No tasks showing
Solution: Verify database was imported, check table name is "tasks"

Problem: Styling looks wrong (no Bootstrap)
Solution: Check internet connection (Bootstrap is loaded from CDN)

Problem: Blank white page
Solution: Check PHP errors in browser console or server logs

IMPORTANT SECURITY NOTE
=======================
